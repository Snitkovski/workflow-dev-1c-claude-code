# Каталог анти-паттернов 1С

> Применяется к: `**/*.bsl`, `**/*.os`

## Система приоритетов

| Приоритет | Влияние | Действие |
|-----------|---------|----------|
| **Critical** | Серьёзная деградация производительности | Исправить немедленно |
| **High** | Заметное влияние на производительность | Исправить в текущей итерации |
| **Medium** | Потенциальные проблемы | Исправить при возможности |
| **Architectural** | Долгосрочные проблемы поддержки | Планировать рефакторинг |

### Формула приоритизации

```
Приоритет = Влияние × Частота × Объём данных
```

## Critical — Критические анти-паттерны

### 1. Запрос в цикле (Query in Loop)

**Влияние:** O(n) обращений к БД вместо O(1)

```bsl
// ❌ НЕПРАВИЛЬНО — N запросов к базе
Для Каждого Строка Из ТаблицаДанных Цикл
    Запрос = Новый Запрос;
    Запрос.Текст = "ВЫБРАТЬ Наименование ИЗ Справочник.Номенклатура ГДЕ Ссылка = &Ссылка";
    Запрос.УстановитьПараметр("Ссылка", Строка.Номенклатура);
    Результат = Запрос.Выполнить();
КонецЦикла;

// ✅ ПРАВИЛЬНО — 1 запрос
МассивСсылок = ТаблицаДанных.ВыгрузитьКолонку("Номенклатура");

Запрос = Новый Запрос;
Запрос.Текст = "ВЫБРАТЬ Ссылка, Наименование ИЗ Справочник.Номенклатура ГДЕ Ссылка В (&СписокСсылок)";
Запрос.УстановитьПараметр("СписокСсылок", МассивСсылок);
Результат = Запрос.Выполнить();
```

---

### 2. Обращение к реквизитам через точку (Dot Notation Access)

**Влияние:** Загрузка всего объекта вместо одного реквизита

```bsl
// ❌ НЕПРАВИЛЬНО — загружает весь объект Контрагент
ИмяКонтрагента = Контрагент.Наименование;
ИННКонтрагента = Контрагент.ИНН;

// ✅ ПРАВИЛЬНО — один запрос только нужных реквизитов
РеквизитыКонтрагента = ОбщегоНазначения.ЗначенияРеквизитовОбъекта(
    Контрагент,
    "Наименование, ИНН");

ИмяКонтрагента = РеквизитыКонтрагента.Наименование;
ИННКонтрагента = РеквизитыКонтрагента.ИНН;

// ✅ ПРАВИЛЬНО — для одного реквизита
ИмяКонтрагента = ОбщегоНазначения.ЗначениеРеквизитаОбъекта(Контрагент, "Наименование");
```

### 3. Подзапрос в SELECT (Subquery in SELECT)

**Влияние:** Выполнение подзапроса для каждой строки (N+1 проблема)

```bsl
// ❌ НЕПРАВИЛЬНО — подзапрос выполняется для каждой строки
Запрос.Текст = "
    |ВЫБРАТЬ
    |   Заказы.Ссылка,
    |   (ВЫБРАТЬ СУММА(Строки.Сумма) ИЗ Документ.Заказ.Товары КАК Строки
    |    ГДЕ Строки.Ссылка = Заказы.Ссылка) КАК ИтогоСумма
    |ИЗ
    |   Документ.Заказ КАК Заказы";

// ✅ ПРАВИЛЬНО — соединение с агрегацией
Запрос.Текст = "
    |ВЫБРАТЬ
    |   Заказы.Ссылка КАК Заказ,
    |   СУММА(Строки.Сумма) КАК ИтогоСумма
    |ИЗ
    |   Документ.Заказ КАК Заказы
    |   ЛЕВОЕ СОЕДИНЕНИЕ Документ.Заказ.Товары КАК Строки
    |   ПО Заказы.Ссылка = Строки.Ссылка
    |СГРУППИРОВАТЬ ПО
    |   Заказы.Ссылка";
```

## High — Высокоприоритетные анти-паттерны

### 4. Фильтрация виртуальной таблицы в WHERE

**Влияние:** Полное сканирование вместо использования индексов

```bsl
// ❌ НЕПРАВИЛЬНО — фильтрация после расчёта виртуальной таблицы
Запрос.Текст = "
    |ВЫБРАТЬ * ИЗ РегистрНакопления.ТоварыНаСкладах.Остатки()
    |ГДЕ Склад = &Склад";

// ✅ ПРАВИЛЬНО — фильтрация в параметрах виртуальной таблицы
Запрос.Текст = "
    |ВЫБРАТЬ * ИЗ РегистрНакопления.ТоварыНаСкладах.Остатки(, Склад = &Склад)";
```

---

### 5. Отсутствие ПЕРВЫЕ N (Missing TOP N)

**Влияние:** Загрузка всех записей когда нужна только часть

```bsl
// ❌ НЕПРАВИЛЬНО — загружает все записи
Запрос.Текст = "ВЫБРАТЬ * ИЗ Справочник.Номенклатура";

// ✅ ПРАВИЛЬНО — ограничение выборки
Запрос.Текст = "ВЫБРАТЬ ПЕРВЫЕ 100 * ИЗ Справочник.Номенклатура";
```

---

### 6. Избыточные вызовы клиент-сервер

**Влияние:** Сетевые задержки и сериализация контекста

```bsl
// ❌ НЕПРАВИЛЬНО — 3 серверных вызова
&НаКлиенте
Процедура ОбработатьДанные()
    Данные1 = ПолучитьДанные1НаСервере();
    Данные2 = ПолучитьДанные2НаСервере();
    Данные3 = ПолучитьДанные3НаСервере();
КонецПроцедуры

// ✅ ПРАВИЛЬНО — 1 серверный вызов
&НаКлиенте
Процедура ОбработатьДанные()
    ВсеДанные = ПолучитьВсеДанныеНаСервере();
КонецПроцедуры

&НаСервереБезКонтекста
Функция ПолучитьВсеДанныеНаСервере()
    Результат = Новый Структура;
    Результат.Вставить("Данные1", ПолучитьДанные1());
    Результат.Вставить("Данные2", ПолучитьДанные2());
    Результат.Вставить("Данные3", ПолучитьДанные3());
    Возврат Результат;
КонецФункции
```

---

### 7. &НаСервере вместо &НаСервереБезКонтекста

**Влияние:** Избыточная передача контекста формы

```bsl
// ❌ НЕПРАВИЛЬНО — передаётся весь контекст формы
&НаСервере
Функция ПолучитьДанныеТовара(Товар)
    Возврат ОбщегоНазначения.ЗначенияРеквизитовОбъекта(Товар, "Наименование, Цена");
КонецФункции

// ✅ ПРАВИЛЬНО — контекст формы не нужен
&НаСервереБезКонтекста
Функция ПолучитьДанныеТовара(Товар)
    Возврат ОбщегоНазначения.ЗначенияРеквизитовОбъекта(Товар, "Наименование, Цена");
КонецФункции
```

**Правило:**
- `&НаСервере` — только когда нужно изменить реквизиты формы
- `&НаСервереБезКонтекста` — для всех остальных серверных операций

## Medium — Среднеприоритетные анти-паттерны

### 8. Отсутствие кеширования в цикле

**Влияние:** Повторные дорогие операции

```bsl
// ❌ НЕПРАВИЛЬНО — повторные вызовы для одинаковых значений
Для Каждого Строка Из ТаблицаДанных Цикл
    ДанныеКонтрагента = ПолучитьДанныеКонтрагента(Строка.Контрагент);
    // Обработка...
КонецЦикла;

// ✅ ПРАВИЛЬНО — кеширование через Соответствие
КешКонтрагентов = Новый Соответствие;

Для Каждого Строка Из ТаблицаДанных Цикл
    ДанныеКонтрагента = КешКонтрагентов.Получить(Строка.Контрагент);

    Если ДанныеКонтрагента = Неопределено Тогда
        ДанныеКонтрагента = ПолучитьДанныеКонтрагента(Строка.Контрагент);
        КешКонтрагентов.Вставить(Строка.Контрагент, ДанныеКонтрагента);
    КонецЕсли;

    // Обработка...
КонецЦикла;
```

---

### 9. Алгоритм O(n²) — вложенные циклы

**Влияние:** Экспоненциальная деградация при росте данных

```bsl
// ❌ НЕПРАВИЛЬНО — O(n²) сложность
Для Каждого Заказ Из Заказы Цикл
    Для Каждого Товар Из Товары Цикл
        Если Заказ.Товар = Товар.Ссылка Тогда
            // Обработка...
        КонецЕсли;
    КонецЦикла;
КонецЦикла;

// ✅ ПРАВИЛЬНО — O(n) через Соответствие
ИндексТоваров = Новый Соответствие;
Для Каждого Товар Из Товары Цикл
    ИндексТоваров.Вставить(Товар.Ссылка, Товар);
КонецЦикла;

Для Каждого Заказ Из Заказы Цикл
    Товар = ИндексТоваров.Получить(Заказ.Товар);
    Если Товар <> Неопределено Тогда
        // Обработка...
    КонецЕсли;
КонецЦикла;
```

---

### 10. Глубокая вложенность (Deep Nesting)

**Влияние:** Плохая читаемость и сопровождаемость

```bsl
// ❌ НЕПРАВИЛЬНО — 5 уровней вложенности
Если Условие1 Тогда
    Если Условие2 Тогда
        Если Условие3 Тогда
            Если Условие4 Тогда
                Если Условие5 Тогда
                    // Код
                КонецЕсли;
            КонецЕсли;
        КонецЕсли;
    КонецЕсли;
КонецЕсли;

// ✅ ПРАВИЛЬНО — ранний выход
Если НЕ Условие1 Тогда
    Возврат;
КонецЕсли;

Если НЕ Условие2 Тогда
    Возврат;
КонецЕсли;

Если НЕ Условие3 Тогда
    Возврат;
КонецЕсли;

Если Условие4 И Условие5 Тогда
    // Код
КонецЕсли;
```

**Правило:** Максимум 3-4 уровня вложенности.

## Архитектурные анти-паттерны

### 11. Big Ball of Mud — отсутствие структуры

**Признаки:**
- Код без разделения на области
- Смешивание бизнес-логики и UI
- Отсутствие модулей

**Решение:** Использовать структуру модулей с областями:
- `ПрограммныйИнтерфейс`
- `СлужебныйПрограммныйИнтерфейс`
- `СлужебныеПроцедурыИФункции`

---

### 12. God Module — модуль-бог

**Признаки:**
- Модуль > 2000 строк
- Модуль отвечает за всё
- Высокая связанность

**Решение:** Декомпозиция на специализированные модули по ответственности.

---

### 13. Tight Coupling — жёсткая связанность

**Признаки:**
- Прямые вызовы внутренних функций
- Зависимость от деталей реализации
- Невозможность изменить один модуль без другого

**Решение:**
- Использовать публичный API (экспортные функции)
- Dependency Injection через параметры
- Программные интерфейсы БСП

---

### 14. Copy-Paste Architecture — копирование кода

**Признаки:**
- Дублирование блоков кода
- Похожие функции с минимальными различиями
- Изменение в одном месте требует изменений в других

**Решение:**
- Выделение общих функций
- Параметризация различий
- DRY принцип

---

### 15. Premature Optimization — преждевременная оптимизация

**Признаки:**
- Сложный код "на будущее"
- Кеширование без измерения
- Абстракции без необходимости

**Решение:**
- Сначала работающий код
- Измерение производительности
- Оптимизация по данным профилирования

## Оптимизация запросов

### 16. Подзапрос в JOIN

```bsl
// ❌ НЕПРАВИЛЬНО — подзапрос в соединении
|ЛЕВОЕ СОЕДИНЕНИЕ (ВЫБРАТЬ ... ИЗ ...) КАК ВложенныйЗапрос

// ✅ ПРАВИЛЬНО — временная таблица
|ВЫБРАТЬ ... ПОМЕСТИТЬ ВременнаяТаблица ИЗ ...
|;
|ВЫБРАТЬ ... ЛЕВОЕ СОЕДИНЕНИЕ ВременнаяТаблица
```

---

### 17. Соединение с виртуальной таблицей

```bsl
// ❌ НЕПРАВИЛЬНО — соединение напрямую с виртуальной таблицей
|ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.Остатки() КАК Остатки

// ✅ ПРАВИЛЬНО — сначала во временную таблицу
|ВЫБРАТЬ ... ПОМЕСТИТЬ Остатки ИЗ РегистрНакопления.Остатки()
|;
|ВЫБРАТЬ ... ЛЕВОЕ СОЕДИНЕНИЕ Остатки
```

---

### 18. Множественные OR в WHERE

**Влияние:** Невозможность использования индексов

```bsl
// ❌ НЕПРАВИЛЬНО — индекс не используется
|ГДЕ Статус = &Статус1 ИЛИ Статус = &Статус2 ИЛИ Статус = &Статус3

// ✅ ПРАВИЛЬНО — использовать В ()
|ГДЕ Статус В (&СписокСтатусов)

// Или ОБЪЕДИНИТЬ для сложных условий
|ВЫБРАТЬ ... ГДЕ Статус = &Статус1
|ОБЪЕДИНИТЬ ВСЕ
|ВЫБРАТЬ ... ГДЕ Статус = &Статус2
```

---

### 19. Разыменование составного типа

**Влияние:** Запросы ко всем возможным типам

```bsl
// ❌ НЕПРАВИЛЬНО — составной тип без указания типа
|ВЫБРАТЬ Документ.Контрагент.Наименование  // Контрагент — составной тип

// ✅ ПРАВИЛЬНО — указание конкретного типа
|ВЫБРАТЬ ВЫРАЗИТЬ(Документ.Контрагент КАК Справочник.Контрагенты).Наименование
```

---

### 20. Отсутствие индекса по фильтруемым полям

**Влияние:** Полное сканирование таблицы

**Решение:**
- Создавать индексы для часто используемых фильтров
- Порядок полей в индексе должен соответствовать запросу
- Использовать функцию `ПРЕДСТАВЛЕНИЕ()` для отображения без соединений

## Запросы — дополнительные правила

### 21. Цепочки вызовов методов результата запроса

**Влияние:** Затрудняет отладку

```bsl
// НЕПРАВИЛЬНО — цепочка вызовов
Выборка = Запрос.Выполнить().Выбрать();

// ПРАВИЛЬНО — промежуточная переменная
РезультатЗапроса = Запрос.Выполнить();
Выборка = РезультатЗапроса.Выбрать();
```

---

### 22. Отсутствие префикса у временных таблиц

**Влияние:** Неочевидность назначения таблицы

```bsl
// НЕПРАВИЛЬНО
|ПОМЕСТИТЬ ДанныеЗаказов

// ПРАВИЛЬНО — префикс ВТ_
|ПОМЕСТИТЬ ВТ_ДанныеЗаказов
```

---

### 23. Конкатенация строк в параметрах запроса

**Влияние:** SQL-инъекция, проблемы производительности

```bsl
// НЕПРАВИЛЬНО — конкатенация
Запрос.Текст = "ВЫБРАТЬ * ГДЕ Наименование ПОДОБНО """ + Строка + "%""";

// ПРАВИЛЬНО — параметр
Запрос.Текст = "ВЫБРАТЬ * ГДЕ Наименование ПОДОБНО &ШаблонИмени";
Запрос.УстановитьПараметр("ШаблонИмени", Строка + "%");
```

## Code Smells — запахи кода

| Запах | Проблема | Решение |
|-------|----------|---------|
| **Data Clumps** | Одни и те же 3+ параметра повторяются | Объединить в Структуру |
| **Primitive Obsession** | Строки вместо перечислений, коды вместо ссылок | `Перечисление`, `СправочникСсылка`, `ОпределяемыйТип` |
| **Divergent Change** | Один модуль меняется по разным причинам | Разделить по ответственности |
| **Shotgun Surgery** | Одно изменение = правки в 5+ местах | Консолидировать в общий модуль |
| **Feature Envy** | Метод интенсивно использует данные другого объекта | Переместить в модуль того объекта |
| **Variable Reuse** | Одна переменная для разных значений | Отдельная переменная на каждое значение |

## Система уверенности при ревью

| Уверенность | Серьёзность | Действие |
|-------------|-------------|----------|
| 90-100 | Critical | Исправить обязательно — блокирует merge |
| 75-89 | High/Medium | Исправить — рекомендуется |
| 50-74 | Architectural | Включать только в архитектурное ревью |
| <50 | Low | **НЕ ВКЛЮЧАТЬ В ОТЧЁТ** |

**Правило:** Для code review сообщать только о проблемах с confidence ≥ 75%. Для архитектурного ревью — ≥ 50%.

## Чек-лист проверки

### Перед code review

- [ ] Нет запросов в циклах
- [ ] Нет обращений через точку к реквизитам ссылок
- [ ] Нет подзапросов в SELECT
- [ ] Виртуальные таблицы фильтруются через параметры
- [ ] Используется `ПЕРВЫЕ N` где применимо
- [ ] Серверные вызовы минимизированы
- [ ] `&НаСервереБезКонтекста` где возможно
- [ ] Кеширование в циклах
- [ ] Нет O(n²) алгоритмов
- [ ] Вложенность ≤ 4 уровней
- [ ] Нет цепочек вызовов (промежуточные переменные)
- [ ] Временные таблицы с префиксом `ВТ_`
- [ ] Нет магических чисел
- [ ] Нет венгерской нотации
- [ ] Нет конкатенации строк в запросах

### При оптимизации

1. **Измерить** текущую производительность
2. **Найти** узкие места (профилирование)
3. **Исправить** по приоритету (Critical → High → Medium)
4. **Измерить** улучшение
5. **Документировать** изменения
